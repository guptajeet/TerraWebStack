Important Terraform Commands

terraform init

Initialize a Terraform working directory. This command downloads and installs providers defined in the configuration, initializes the backend, and downloads modules.

terraform plan

Generate and show an execution plan. This lets you preview the changes Terraform plans to make to your infrastructure.

terraform apply

Builds or changes infrastructure according to your Terraform configuration files. By default, it will ask for confirmation before applying changes.

terraform destroy

Destroys all remote objects managed by a particular Terraform configuration. Use with caution!

terraform fmt

Rewrites Terraform configuration files to a canonical format and style. This command helps maintain consistent formatting across your Terraform files.

terraform validate

Validates the syntax and internal consistency of Terraform configuration files in a directory, referring only to the configuration and not accessing any remote services.

Key Terraform Concepts

Infrastructure as Code (IaC)

Managing and provisioning infrastructure through code instead of manual processes. This approach allows for version control, reproducibility, and automation of infrastructure management.

State Management

How Terraform keeps track of the current state of your infrastructure. The state file is crucial for determining which changes to make to your infrastructure when you update your configuration.

Provider

A plugin that allows Terraform to interact with various cloud providers and services. Providers are responsible for understanding API interactions and exposing resources.

Resource

The most important element in the Terraform language. Each resource block describes one or more infrastructure objects, such as virtual networks, compute instances, or higher-level components such as DNS records.

Data Source

Allows Terraform to use information defined outside of Terraform, defined by another separate Terraform configuration, or modified by functions.
